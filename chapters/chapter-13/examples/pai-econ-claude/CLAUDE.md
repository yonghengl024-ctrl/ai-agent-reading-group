## ⚠️ CRITICAL: No Unverified Citations — Ever

**LLMs hallucinate academic citations. This is a known, serious failure mode.**

A hallucinated citation looks exactly like a real one: plausible author names,
a real-sounding title, a real journal, a plausible year and volume. The details
that are wrong (page numbers, the actual title, whether the paper exists at all)
are the details you cannot check from training data alone.

### The rule

**Never write a citation into any output file — draft, skeleton, or final PDF —
unless it has been confirmed by an external web search in the current session.**

This includes:
- `literature_positioning.md` (Stage 2)
- `candidate_propositions.md` (Stage 6 — "Connection to Prior Literature" sections)
- `economic_interpretation.md` (Stage 9)
- `manuscript.tex` / `manuscript.pdf` (Stage 10)
- Any response to the user that quotes a specific paper

### How to verify

For each citation you intend to include, use WebSearch or WebFetch to confirm:
1. The paper exists with the exact title you are attributing to it
2. The author names and year are correct
3. The journal, volume, issue, and page numbers match

Acceptable sources: Google Scholar, JSTOR, NBER, SSRN, journal websites,
publisher DOI pages. A result from one of these sources that matches all fields
is a VERIFIED citation.

If you cannot find a confirmation: mark the citation UNVERIFIED and exclude it
from the output. Tell the user which citations could not be verified.

### What is safe without web search

The following landmark papers are so well-established that their core details
(author, year, venue) can be included without a fresh web search, but **page
numbers and volume/issue numbers must still be verified**:

- Becker (1964) *Human Capital* — NBER/University of Chicago Press
- Arrow (1973) "The Theory of Discrimination" — in Ashenfelter & Rees (eds.),
  Princeton University Press
- Galor & Zeira (1993) "Income Distribution and Macroeconomics" — RES 60(1): 35–52

Everything else — especially applied empirical papers, China-specific literature,
and any paper you are less than 100% certain about — requires web verification.

### How this failure happened

In Project_002_BeckerHukou, the pipeline wrote `manuscript.tex` with 7 citations.
Only 3 were verified (Arrow, Becker, Galor-Zeira). Stage 2 had explicitly flagged
Chan & Zhang (1999) as UNCERTAIN with the note "Do not cite without verification."
Two additional citations (Lu & Song 2006, Meng 2012) were added from training
data alone with no pipeline backing. All three were removed before the final PDF.

## PDF Output Standard

Final PDFs for theoretical framework papers follow the style of
`260311_genetic_diversity_skill_formation_model_v3.pdf`. Key rules:

- Write a complete `.tex` file; compile with `pdflatex` (not pandoc)
- `\documentclass[12pt]{article}` — Computer Modern font, standard margins
- Title format: `"<brief problem description (≤8 words)>: A Theoretical Framework"`
  — fits in 2 lines; use `\Large\bfseries` via `\makeatletter` patch (not default `\LARGE`)
- Author: `\texttt{pAI-Econ-claude} (\texttt{theoretical-economics-claude-skill})`
  then GitHub URL then `\small Claude Sonnet 4.6` (no "Generated by", not italic)
- Green hyperlinks: `\usepackage{xcolor}`, `\definecolor{linkgreen}{RGB}{0,120,100}`,
  load `hyperref` with `colorlinks=true, linkcolor=linkgreen, citecolor=linkgreen`
- `thebibliography` handles the "References" header automatically — do NOT add
  a separate `\section*{References}` (creates a duplicate header)
- Only include VERIFIED citations in the bibliography

See `memory/feedback-pdf-style.md` for complete style rules.
